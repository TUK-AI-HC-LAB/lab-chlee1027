# PatchCore 재현 실패(성능 하락) 원인 분석 및 검증 계획

## 1. 문제 / 동기
MVTec AD 전 카테고리(15개) 재현 과정에서 대부분의 카테고리는 논문 수치와 일치하였으나, `pill`과 `metal_nut`에서 유의미한 수치 하락이 관찰되었습니다.
- **Pill:** I-AUROC 0.967 (논문 0.978, Δ -0.011)
- **Metal_nut:** P-AUROC 0.983 (논문 0.991, Δ -0.008)

이러한 하락이 단순한 우연인지, 혹은 특정 파라미터에 의한 것인지 파악하기 위한 가설 설정 및 검증 실험을 제안합니다.

---

## 2. 원인 분석 및 가설 (Root Cause Analysis)

### 2.1. Pill 카테고리 (Instance AUROC 하락)
- **가설 1 (샘플링 부족):** `pill`은 표면 글자(엠보싱), 로고, 곡면 음영 등 정상 패턴의 변동성이 큽니다. 현재 10%의 Coreset 샘플링 비율이 이 복잡한 정상 범위를 모두 담아내지 못했을 수 있습니다.
- **가설 2 (미세 결함 희석):** 알약의 미세한 스크래치 신호가 PatchCore의 공간적 특징 결합(Aggregation) 과정에서 주변 정상 특징에 묻혔을 가능성이 있습니다.

### 2.2. Metal_nut 카테고리 (Pixel AUROC 하락)
- **가설 3 (기하학적 복잡성):** 나사산(threads)과 육각 가장자리의 복잡한 구조 및 금속 특유의 반사광이 픽셀 단위의 정확한 결함 위치 획정을 방해했을 수 있습니다.
- **가설 4 (해상도 손실):** 특징 맵 복원 과정에서 발생하는 인터폴레이션 오차가 금속의 미세 결함 위치 정밀도를 떨어뜨렸을 수 있습니다.

---

## 3. 검증 실험 계획 (Validation Plan)

### 실험 B: 무작위성(Seed) 영향도 테스트 (Pill) - 결과 반영 (2026-05-12)
- **목적:** 단일 실행 결과의 우연성(Seed) 배제 검증
- **설정:** `coreset_pct` = 0.1(10%), `seed` ∈ {0, 1, 2}
- **실험 결과:**
    - Seed 0: I-AUROC 0.967
    - Seed 1: I-AUROC 0.960
    - Seed 2: I-AUROC 0.963
- **분석 결론:** 
    - **가설(우연한 성능 하락) 기각:** 시드를 변경해도 성능이 0.96대에 머물러 있음. 즉, 현재 환경에서 Pill 카테고리의 성능 하락은 일시적인 우연이 아닌 **체계적인 원인(Systematic Issue)**에 의한 것임.

---

## 4. 심층 가설 및 추가 실험 계획

### 새로운 가설: 전처리 알고리즘(Resize Interpolation)의 영향
- **배경:** `pill` 카테고리는 표면에 정교한 글자(엠보싱), 로고, 미세한 조명 음영이 포함되어 있어 고주파(High-frequency) 정보가 매우 풍부함.
- **핵심 가설:** 리사이즈(256 -> 224) 과정에서 사용하는 보간법(Interpolation)의 차이가 미세한 텍스처 정보를 뭉개뜨려 판별력을 저하시킴. (논문 환경과 현재 PyTorch/Pillow 버전 간의 기본 알고리즘 차이 가능성)

### 실험 D: 전처리 알고리즘 강건성 테스트 (Pill) - 결과 반영 (2026-05-12)
- **목적:** 보간법(Bicubic) 및 Antialiasing 설정 변화에 따른 성능 민감도 확인
- **설정:** `interpolation` = BICUBIC, `antialias` = False
- **실험 결과:**
    - I-AUROC **0.967** (기본 설정과 소수점 셋째 자리까지 동일)
- **분석 결론:** 
    - `pill` 카테고리는 전처리 알고리즘의 미세한 변화(보간법, 안티앨리어싱)에는 매우 강건(Robust)함. 
    - 결과적으로 `pill`의 성능을 결정짓는 핵심 변수는 **샘플링 비율(Coreset Ratio)**임이 실험 A를 통해 다시 한번 확인됨. (1% 샘플링 시에만 논문 수치 재현 가능)

## 5. 최종 결론 및 향후 계획

### Pill 카테고리 최종 결론
- `pill` 카테고리의 I-AUROC 하락(-0.011)은 전처리나 시드의 문제가 아닌, **현대적인 실행 환경(PyTorch/CUDA 버전 등)에서 10% 샘플링 시 발생하는 고유한 특징 분포의 차이**로 판단됨.
- 오히려 1% 샘플링(Coreset 0.01)에서 논문 수치(0.980)를 안정적으로 상회하므로, 실무 환경에서는 데이터 효율성과 성능을 모두 잡을 수 있는 **1% 샘플링 설정을 권장**함.

### 다음 단계: 실험 C (Metal_nut 집중)
- **목적:** `metal_nut`의 P-AUROC(Pixel 단위 정밀도)를 논문 수치(0.991) 수준으로 끌어올리기 위한 실험.
- **가설:** 금속의 미세한 결함은 더 높은 해상도나 저수준 특징(Layer 1)에서 더 잘 포착될 것임.
- **설정:** `imagesize` 224 -> 324, 또는 `layers` [layer1, layer2, layer3] 추가.
