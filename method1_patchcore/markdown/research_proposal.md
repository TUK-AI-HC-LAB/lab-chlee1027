# [Refined] PatchCore 한계 극복 및 Method 1~4 통합 연구 제안서 (v3)

본 문서는 PatchCore, SimpleNet, RD, Dinomaly의 재현 및 비교 분석 결과를 토대로, 각 알고리즘의 핵심 메커니즘을 융합하여 기존 PatchCore의 한계를 보완하고 실무 배포성을 향상시키기 위한 후속 연구 방향을 제안함.

---

## 1. [주제 A] 하드웨어 제약 기반 고속·고효율 패치코어 최적화 (High-Speed & Memory-Efficient PatchCore via ANN)

### 1.1 개요 및 동기
- **PatchCore (M1)의 약점**: 로컬 패치 특징을 고차원($d=1024$) 그대로 메모리 뱅크에 수만 개 이상 저장하고 Exact Search(`IndexFlatL2`)로 조회하므로, 데이터 크기가 커질수록 **검색 시간(Latency)이 급증하고 인덱스 메모리 점유율이 팽창**하여 실제 산업용 엣지 디바이스 배포에 한계가 있음.
- **통합 최적화 아이디어**: 메모리 뱅크 전단에 근사 최근접 이웃 검색(Approximate Nearest Neighbor, ANN)인 **FAISS 기법(IVF/HNSW/PQ)**을 도입하고 하이퍼파라미터를 튜닝하여, 정확도 손실을 최소화하면서 연산 속도와 메모리를 극대화하는 성능-속도-용량의 다차원 트레이드오프 파레토 최적(Pareto Frontier)을 설계함.

### 1.2 문제 정의 (Problem Definition)

먼저 연구에 필요한 핵심 객체들을 고정한 뒤, 풀고자 하는 최적화 문제를 정의합니다.

정의 1. 패치코어 추론 단계별 컴포넌트 (PatchCore Inference Components).
패치코어 추론 파이프라인은 특징 추출 ($T_{\text{feat}}$), 최근접 이웃 검색 ($T_{\text{search}}$), 후처리 ($T_{\text{post}}$)의 연속적인 세 단계로 구성된다. 이 중 고차원 공간($d=1024$)에서의 exact 최근접 이웃 검색 단계가 테스트 과정의 핵심 연산 병목을 형성한다.

정의 2. 근사 최근접 이웃 (ANN) 인덱스 $I(\Theta)$.
검색 질의 속도를 높이기 위해 coreset 특징 집합 $M_c$ 위에 구축되는 인덱스 구조(예: FAISS `IVF`, `HNSW`, `PQ` 등)이며, 검색 하이퍼파라미터 설정 $\Theta$(예: 센트로이드 개수 $n_{list}$, 검색 프로브 수 $n_{probe}$, 노드 연결 수 $M$ 등)에 의해 매개변수화된다.

문제 1. 이상 탐지 성능과 추론 효율성의 공동 최적화.
타겟 테스트 이미지 $x$가 주어졌을 때, 우리의 태스크는 $M_c$ 위에 구축된 ANN 인덱스 $I(\Theta)$를 사용하여 이상 점수 맵 $S(x)$를 계산하는 것이다. 구체적으로, 우리는 다음 제약 조건 하에서 최적의 인덱스 기법 $I \in \{\text{IVF}, \text{HNSW}, \text{PQ}\}$와 검색 파라미터 $\Theta$를 결정해야 한다:
  - 최대 인덱스 메모리 용량 제약: $\text{Memory}(I(\Theta)) \le M_{\max}$
  - 추론 검색 시간(Latency) 제약: $T_{\text{search}}(I(\Theta)) \le T_{\max}$
최종 목표는 검색 속도(Latency)와 메모리 사용량을 최소화하면서도 이상 탐지 성능(I-AUROC, P-AUROC)을 최대화하는 것이다.

| 항목 | 질문 | 내용 |
|---|---|---|
| **입력** | 무엇이 주어지는가? | Coreset 정상 패치 메모리 뱅크 $M_c$, 테스트 이미지 패치 특징 $P_T$ |
| **출력** | 무엇을 예측해야 하는가? | 최적화된 검색 인덱스 기법 $I$ 및 탐색 하이퍼파라미터 $\Theta$ |
| **단위** | 처리 단위가 무엇인가? | 패치 특징 차원 $d$ 및 테스트 패치 1개당 탐색 거리 측정 |
| **제약** | 제약 조건이 무엇인가? | 최대 허용 검색 시간 $T_{\max}$ 및 인덱스 점유 메모리 한계 $M_{\max}$ |
| **성공 기준** | 성공 조건이 무엇인가? | Exact search(IndexFlatL2) 대비 성능 하락(AUROC) $\le 1\%$ 이내를 유지하면서 속도 5배 이상 개선 |

### 1.3 관련 연구 대비 차별성
| 구분 | 기존 연구 (Memory compression variants) | 본 제안 방식 (Unified Curation) |
| :--- | :--- | :--- |
| **접근** | 수학적 압축 (PQ, 차원 축소) 등 정적 최적화 중심 | **M4의 통합 학습 및 M3의 정보 변별력**을 활용한 동적 관리 |
| **장점** | 저장 용량 감소 | Continual AD 환경에서 성능 하락을 최소화하며 신규 정보 수용력 확보 |

### 1.4 기대 효과 및 한계
- **장점**: 즉각적인 실무 배포가 가능한 고속 패치코어를 획득할 수 있으며, 엣지 머신에서의 메모리/시간 비용을 혁신적으로 절감함.
- **단점**: Quantization 및 Approximation 과정에서 발생하는 불가피한 극미세 결함 탐지 성능 하락을 방지하는 세부 튜닝이 필수적임.

## 2. [주제 B] 관계 추론 및 Attention 기반 논리적 이상 탐지 (Contextual Logical AD)

### 2.1 개요 및 동기
- **PatchCore (M1)의 약점**: 로컬 패치 특징에 의존하는 구조적 특성상, 부품의 구성 요소는 정상이지만 배치 관계나 개수가 어긋난 '논리적 결함(Logical Anomaly)' 탐지에 한계가 있음.
- **Dinomaly (M4)의 강점**: **Linear Attention**을 통해 이미지 전반의 전역적 문맥 정보를 파악하는 데 유리함.
- **통합 아이디어**: PatchCore의 메모리 뱅크 체계에 Dinomaly의 **Attention-based contextual encoding을 결합**. 패치 특징과 함께 Attention Map 기반의 **"Relational Context"**를 결합 저장하여, 공간적 위상 관계 이상을 포착함.

### 2.2 문제 정의 (Problem Definition)

먼저 연구에 필요한 핵심 관계 객체들을 정의한 뒤, 논리적 이상 탐지 문제를 정의합니다.

정의 1. 로컬 패치 특징 및 관계 그래프 $G = (V, E)$.
$V$는 백본 네트워크로부터 추출된 로컬 패치 특징 집합 $\{p_i\}$이다. $E$는 공간적 및 의미론적 관계 에지 집합 $e_{ij} = (p_i, p_j)$이며, DINOv2 self-attention 맵 또는 다중 스케일 특징에 의해 가중치가 부여된다.

정의 2. 관계적 문맥 기술자 (Relational Context Descriptor) $c_i$.
로컬 패치 특징 $p_i$와 DINOv2 linear attention 매핑을 활용한 이웃 패치들 간의 공간적·의미론적 관계를 결합하여 구성된 patch $i$의 문맥 기술자이며, $c_i = \text{Concat}(p_i, \text{Attn}(p_i))$로 정의된다.

문제 1. 논리적 및 문맥적 이상 탐지.
테스트 이미지 $x$가 주어졌을 때, 우리의 태스크는 논리적 이상 점수 $S_{\text{logical}}(x)$를 계산하는 것이다. 구체적으로, 우리는 오직 정상 데이터만 주어지는 zero-shot/few-shot 학습 제약 조건 하에서, 객체 구성 부품들의 공간적 배치 패턴, 정렬 방향, 개수가 정상 분포로부터 벗어나는지 여부를 결정/예측해야 한다. 최종 목표는 논리적 이상 탐지 정확도를 최대화하는 것이다.

| 항목 | 질문 | 내용 |
|---|---|---|
| **입력** | 무엇이 주어지는가? | 정상 이미지 패치 특징 그래프 $G_N$, 테스트 이미지 패치 특징 그래프 $G_T$ |
| **출력** | 무엇을 예측해야 하는가? | 논리적 결함 판별 점수 $S_{\text{logical}}(x)$ |
| **단위** | 처리 단위가 무엇인가? | 패치 관계 에지(Edge) 및 픽셀/패치 레벨 위상 구조 |
| **제약** | 제약 조건이 무엇인가? | 정상 데이터만으로 관계적 정상 분포 모델링 필요 (One-class classification) |
| **성공 기준** | 성공 조건이 무엇인가? | MVTec LOCO AD 데이터셋의 Logical Anomaly 부문에서 기존 PatchCore 대비 AUROC 5% 이상 성능 향상 |

### 2.3 관련 연구 대비 차별성
| 기존 연구 | 주요 특징 | 본 제안 방식의 차별점 (Contribution) |
| :--- | :--- | :--- |
| **PNI** (ICCV '23) | 패치의 절대 좌표 정보를 MLP로 학습 | 절대 좌표 의존성 탈피 및 **M4의 Attention 기반 관계 특징** 활용 |
| **GraphCore** (ICLR '23) | Rotation-invariant feature representation 중심 | **Logical anomaly** 탐지를 위한 relational context reasoning에 집중 |

### 2.4 기대 효과 및 한계
- **장점**: MVTec LOCO AD 등 최신 벤치마크 대응에 유리하며, 설명 가능한(Explainable) AD로의 확장성이 높음.
- **단점**: 패치 간 관계 데이터(Edge) 추가에 따른 메모리 사용량 및 연산량 최적화가 필요함.

## 3. [주제 C] 도메인 적응형 동적 패치 시스템 (Domain-adaptive Patching)

### 3.1 개요 및 동기
- **PatchCore (M1)의 약점**: ImageNet 사전학습 모델의 표현력에 의존하여 산업 데이터와의 도메인 갭 발생 및 영역별 연산 낭비 발생.
- **SimpleNet (M2)의 강점**: **Feature Adapter**를 통한 산업 도메인 특징 공간으로의 효과적 정렬 능력.
- **통합 아이디어**: PatchCore 전단에 SimpleNet 스타일의 **Lightweight Adapter**를 결합하여 도메인 적응을 수행함과 동시에, 영역별 정보 엔트로피에 따라 패치 해상도를 동적으로 조절함(복잡 영역: 정밀, 단순 영역: 성기게 처리).

### 3.2 문제 정의 (Problem Definition)

먼저 연구에 필요한 도메인 정렬 및 동적 스케일 관련 객체들을 정의한 뒤, 제약조건 하의 이상 탐지 문제를 정의합니다.

정의 1. 도메인 어댑터 $A_\theta$ 및 엔트로피 가이드 선택.
$A_\theta$는 ImageNet 사전학습 특징을 타겟 산업 이미지 도메인에 정렬하기 위해 학습되는 경량 특징 투영 어댑터이다. $H(x_i)$는 타겟 이미지의 공간적 특징 엔트로피 맵으로, 복잡한 구조를 가진 영역과 균일한 텍스처 영역을 식별해 낸다.

정의 2. 다중 해상도 동적 패칭 연산자 $\Psi$.
고엔트로피/복잡 구조 영역에는 높은 패치 해상도(정밀 샘플링)를 할당하고, 저엔트로피/단순 텍스처 영역에는 낮은 패치 해상도(성긴 샘플링)를 동적으로 배정하는 연산자이다.

문제 1. 자원 제약 하의 도메인 적응형 이상 탐지.
테스트 이미지 $x$가 주어졌을 때, 우리의 태스크는 엄격한 연산량 예산 제약(FLOPs/시간 한계) 하에서 도메인 적응된 특징 $A_\theta(f)$를 기반으로 도메인 적응형 아노말리 맵 $S_{\text{adapted}}(x)$를 계산하는 것이다. 최종 목표는 타겟 도메인에서의 이상 탐지 성능을 극대화하는 것이다.

| 항목 | 질문 | 내용 |
|---|---|---|
| **입력** | 무엇이 주어지는가? | 타겟 산업 이미지 특징 맵 $f$, 어댑터 파라미터 $\theta$, 로컬 특징 엔트로피 맵 $H$ |
| **출력** | 무엇을 예측해야 하는가? | 도메인 적응형 동적 패치 아노말리 맵 $S_{\text{adapted}}(x)$ |
| **단위** | 처리 단위가 무엇인가? | 영역별 동적 해상도 패치 (Dynamic multi-scale patch) |
| **제약** | 제약 조건이 무엇인가? | 추가 학습 계층의 연산량 및 도메인 적응 어댑터 가중치 용량 제약 |
| **성공 기준** | 성공 조건이 무엇인가? | CFA/SimpleNet 대비 동등 이상의 I-AUROC 성능을 유지하면서 도메인 갭 보완 및 패치 샘플링 연산량(FLOPs) 30% 이상 절감 |

### 3.3 관련 연구 대비 차별성
- 모든 위치에 다중 스케일을 고정 적용하는 기존 방식(CFA 등)과 달리, **도메인 특화 어댑터**가 영역의 중요도를 판단하여 선택적 스케일을 적용함으로써 실무적인 연산 효율성을 극대화함.

### 3.4 기대 효과 및 한계
- **장점**: 도메인 갭으로 인해 발생하는 미세 하락(예: `pill`, `metal_nut`)을 예방하고, 실무 현장에서 불필요한 배경 영역 연산을 획기적으로 줄여 추론 효율을 극대화함.
- **단점**: 특징 엔트로피 기반의 영역 판단 임계값(Threshold) 설정 시 카테고리별 편차가 크게 나타날 수 있음.

## 4. 실험 계획 (Experimental Plan)

### 4.1 대상 데이터셋
- **MVTec AD & VisA**: 주제 A, C의 범용성 및 하드웨어 최적화 검증용 필수 데이터셋.
- **MVTec LOCO AD**: 주제 B(Contextual Logical AD) 검증을 위한 필수 데이터셋.

### 4.2 비교 대상 (Baselines)
- **PatchCore (Original - FlatL2)**: 기본 성능 및 연산 메모리/속도 베이스라인.
- **FAISS Variants (IVF / HNSW / PQ)**: 주제 A의 벤치마크 대상 및 다차원 최적 분석군.
- **EfficientAD, SimpleNet**: 연산 효율 및 도메인 적응 성능 비교군.

### 4.3 평가지표 (Metrics)
- **I-AUROC / P-AUROC**: 탐지 및 위치추정 기본 성능.
- **Memory Footprint (Index Memory / Peak Memory)**: 주제 A, C의 실용적 하드웨어 가치 증명.
- **Search Latency (Inference Time)**: 주제 A의 실무 성능 가용 증명.
- **sAUROC (Structural) / lAUROC (Logical)**: MVTec LOCO 기반의 결함 종류별 탐지 능력.

## 5. 결론 및 건의

본 제안은 직접 재현한 4개 알고리즘(M1~M4)의 공학적 분석을 기반으로 도출된 후속 연구 방향임. 제안된 3가지 주제(A, B, C) 중 연구실의 차기 목표이자 교수님의 단기 과제 지시에 정확히 융합되는 **[주제 A] 하드웨어 제약 기반 고속·고효율 패치코어 최적화**를 최우선으로 진행하여 정량적이고 학술적인 비교 데이터를 우선 확보하고자 함. 교수님의 검토 및 지도를 요청함.
